== Internal anchors and cross-references

`[#id]` adds an anchor to a heading, and `<<id,text>>` references it. Clicking scrolls to the spot without switching pages.

*Source:*

[source,asciidoc]
------
[#install-section]
=== Installation steps (with an anchor)

This heading has an anchor (ID) via `[#install-section]`.
You can reference it from elsewhere like <<install-section,go to the installation steps>>.
------

*Rendered:*

[#install-section]
=== Installation steps (with an anchor)

This heading has an anchor (ID) via `[#install-section]`.
You can reference it from elsewhere like <<install-section,go to the installation steps>>.

== Images

`image::` is a block image (with a caption); `image:` is an inline image (images are embedded into the single HTML as data URIs).

*Source:*

[source,asciidoc]
------
.monodocs logo
image::images/logo.svg[Logo, 120]

You can also place an inline image image:images/logo.svg[Logo, 80] within text.
------

*Rendered:*

.monodocs logo
image::images/logo.svg[Logo, 120]

You can also place an inline image image:images/logo.svg[Logo, 80] within text.

== Footnotes

`footnote:[...]` creates a footnote; `footnote:ref1[...]` creates a reusable named footnote.

*Source:*

[source,asciidoc]
------
You can attach a footnote to the textfootnote:[This is a footnote. It gets a unique ID within the single HTML.].
You can also use a named footnotefootnote:ref1[A reusable footnote.] and reference the same one againfootnote:ref1[].
------
