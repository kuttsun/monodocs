# リスト

各項目は **ソース** を先に示し、その下に **表示（HTML 変換結果）** を並べます。

## 箇条書き（順序なし）

`-` / `*` / `+` のいずれもマーカーに使えます。インデントでネストします。

**ソース:**

```markdown
- 項目 A
- 項目 B
  - ネスト B-1
  - ネスト B-2
    - さらにネスト
- 項目 C
```

**表示:**

- 項目 A
- 項目 B
  - ネスト B-1
  - ネスト B-2
    - さらにネスト
- 項目 C

## 番号付き（順序あり）

開始番号も指定できます。番号リスト同士は間に段落を挟むと別リストになり、2 つ目は指定した番号から始まります（空行だけでは同じリストの続きになります）。

**ソース:**

```markdown
1. 最初
2. 次
3. その次
   1. ネスト 3-1
   2. ネスト 3-2

開始番号を指定（間に段落を挟んで別リストにする）:

5. 5 から
6. 6
7. 7
```

**表示:**

1. 最初
2. 次
3. その次
   1. ネスト 3-1
   2. ネスト 3-2

開始番号を指定（間に段落を挟んで別リストにする）:

5. 5 から
6. 6
7. 7

## タスクリスト（GFM 拡張）

`- [ ]` / `- [x]` でチェックボックスになります。

**ソース:**

```markdown
- [x] 完了したタスク
- [ ] 未完了のタスク
  - [x] 子タスク（完了）
  - [ ] 子タスク（未完了）
```

**表示:**

- [x] 完了したタスク
- [ ] 未完了のタスク
  - [x] 子タスク（完了）
  - [ ] 子タスク（未完了）

## ゆるいリストと詰まったリスト

項目間に空行があると「ゆるい（loose）」リストになり、各項目が段落として描画されます。

**ソース:**

```markdown
詰まった（tight）リスト:

- 行間が詰まる
- 2 行目

ゆるい（loose）リスト（項目間に空行）:

- 段落として描画される

- 項目の間に空行があると緩くなる
```

**表示:**

詰まった（tight）リスト:

- 行間が詰まる
- 2 行目

ゆるい（loose）リスト（項目間に空行）:

- 段落として描画される

- 項目の間に空行があると緩くなる

## 項目に複数ブロックを含める

リスト項目にインデントを揃えて段落や引用を続けられます。

**ソース:**

```markdown
- 項目に複数段落を含む例。

  これは同じ項目の 2 段落目です（インデント）。

  > 項目内の引用。
```

**表示:**

- 項目に複数段落を含む例。

  これは同じ項目の 2 段落目です（インデント）。

  > 項目内の引用。
