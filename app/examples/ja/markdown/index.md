# GitHub Flavored Markdown サンプル

このサンプルは [GitHub Flavored Markdown](https://github.github.com/gfm/)（CommonMark + GFM 拡張）の
主な記法を網羅し、`monodocs` での表示を確認するためのものです。

各項目は **ソース（生の Markdown）** を先に示し、その下に **表示（HTML 変換結果）** を並べます。

ビルド例:

```bash
monodocs build examples/ja -o dist/showcase.html
```

各カテゴリは左サイドバーから開けます:
[テキスト書式](text.md) / [リスト](lists.md) / [コードと表](code-and-tables.md) / [脚注とリンク参照](footnotes.md) / [アラート](alerts.md) / [幅の広いコンテンツ](wide-content.md) / [応用（ネスト階層）](advanced/index.md)。

## 見出し（ATX）

`#` を 1〜6 個でレベルを表します。

**ソース:**

```markdown
## レベル 2

### レベル 3

#### レベル 4

##### レベル 5

###### レベル 6
```

**表示:**

## レベル 2

### レベル 3

#### レベル 4

##### レベル 5

###### レベル 6

## 見出し（Setext）

`===` / `---` の下線でも見出しになります（H1 / H2 のみ）。

**ソース:**

```markdown
大見出し（Setext H1）
===================

小見出し（Setext H2）
-------------------
```

**表示:**

大見出し（Setext H1）
===================

小見出し（Setext H2）
-------------------

## 段落と改行

空行で段落を区切ります。強制改行は行末のバックスラッシュ `\`（または目に見えない空白 2 つ）で行います。
ここでは可視で分かりやすいバックスラッシュの例を示します。

**ソース:**

```markdown
これは段落です。空行で段落を区切ります。

バックスラッシュで強制改行できます。\
このとおり次の行になりました。
```

**表示:**

これは段落です。空行で段落を区切ります。

バックスラッシュで強制改行できます。\
このとおり次の行になりました。

## 水平線

`---` / `***` / `___` のいずれでも水平線になります。

**ソース:**

```markdown
セクションの区切り。

---
```

**表示:**

セクションの区切り。

---

## 引用

`>` で引用、ネストや引用内のリストも書けます。

**ソース:**

```markdown
> 引用です。
>
> > ネストした引用。
>
> - 引用内のリスト
> - 2 つ目
```

**表示:**

> 引用です。
>
> > ネストした引用。
>
> - 引用内のリスト
> - 2 つ目

## エスケープと実体参照

バックスラッシュで記号をエスケープし、HTML 実体参照も使えます。

**ソース:**

```markdown
バックスラッシュで記号をエスケープ: \*not italic\* / \`not code\` / \# not heading

実体参照: &copy; &amp; &lt; &gt; &#169; &hearts;
```

**表示:**

バックスラッシュで記号をエスケープ: \*not italic\* / \`not code\` / \# not heading

実体参照: &copy; &amp; &lt; &gt; &#169; &hearts;
