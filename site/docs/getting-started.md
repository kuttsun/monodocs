# Getting Started

> **Status: v0.6 beta candidate** — HTML, PDF, validation, Mermaid, `watch`, and `serve` are implemented. The npm prerelease is being prepared under the `next` tag.

monodocs turns a directory of Markdown / AsciiDoc files into **one self-contained HTML file**. You keep your documentation split across many files; only the distributed artifact is bundled into a single file.

## What it produces

```bash
monodocs build ./docs -o ./dist/manual.html
```

The output is a single HTML with no external runtime: in-page search, a table of contents, prev/next navigation, dark mode, and a print layout are all embedded. Try the **Single-file demo** link in the top navigation — it is generated by monodocs from this project's own examples.

## Install the beta

After the beta is announced on npm, install it with Node.js 22 or later:

```bash
npm install -g monodocs@next
# Or run without a global installation
npx monodocs@next build ./docs -o ./dist/manual.html
```

PDF output and Mermaid pre-rendering require Chromium or Google Chrome. Set `PUPPETEER_EXECUTABLE_PATH` if the browser is not discovered automatically.

## Run from source before the beta is published

Until the npm prerelease is announced, build from source. The development toolchain runs **inside Docker**, so you don't need Node or pnpm on your host:

```bash
git clone https://github.com/kuttsun/monodocs.git
cd monodocs

scripts/app.sh pnpm install
scripts/app.sh pnpm build

# Build a single HTML from the bundled examples
scripts/app.sh node packages/cli/dist/index.js build examples/en -o dist/manual.html
```

## Preview locally

```bash
# Live-reloading preview server
scripts/app.sh node packages/cli/dist/index.js serve examples/en --host 0.0.0.0
# → http://localhost:4173/
```
