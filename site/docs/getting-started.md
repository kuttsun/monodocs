# Getting Started

> **Status: v0.4** — monodocs generates a single HTML from Markdown / AsciiDoc (link conversion, image embedding, Mermaid, in-page search, TOC, dark mode, `watch` / `serve`). PDF output is planned (v0.5); npm distribution is planned (v0.6).

monodocs turns a directory of Markdown / AsciiDoc files into **one self-contained HTML file**. You keep your documentation split across many files; only the distributed artifact is bundled into a single file.

## What it produces

```bash
monodocs build ./docs -o ./dist/manual.html
```

The output is a single HTML with no external runtime: in-page search, a table of contents, prev/next navigation, dark mode, and a print layout are all embedded. Try the **Single-file demo** link in the top navigation — it is generated by monodocs from this project's own examples.

## Run from source

monodocs is not published to a package registry yet, so build it from source. The development toolchain runs **inside Docker**, so you don't need Node or pnpm on your host:

```bash
git clone https://gitlab.com/kuttsun/monodocs.git
cd monodocs

scripts/app.sh pnpm install
scripts/app.sh pnpm build

# Build a single HTML from the bundled examples
scripts/app.sh node packages/cli/dist/index.js build examples/en -o dist/manual.html
```

## Preview locally

```bash
# Live-reloading preview server
scripts/app.sh node packages/cli/dist/index.js serve examples/en --host 0.0.0.0
# → http://localhost:4173/
```
