# monodocs site

[日本語](README.ja.md)

The official monodocs landing page and documentation site. It is built with **VitePress** and produces static output suitable for GitHub Pages or GitLab Pages.

The site is a standalone npm package, separate from the `app/` pnpm workspace. All builds run in Docker, so Node.js and npm are not required on the host.

## Structure

```text
site/
  .vitepress/config.ts   Locales (English at root, Japanese at /ja/), local search, base switching
  index.md               English landing page
  ja/index.md            Japanese landing page
  docs/                  English documentation
  ja/docs/               Japanese documentation
  public/manual.html     Single-HTML demo generated by monodocs (generated, not tracked)
```

## Languages

English is the root and default locale. Every path outside `/ja/` is served in English, and unsupported locales fall back to English.

## Development and build

Start a local development server from the host:

```bash
scripts/site-dev.sh                      # http://localhost:5173/ with HMR
```

Run individual steps with:

```bash
scripts/site.sh npm install              # Install dependencies
scripts/site.sh npm run docs:dev         # Development server at http://localhost:5173/
scripts/site.sh npm run docs:build       # Production build to site/.vitepress/dist/
scripts/site.sh npm run docs:preview     # Preview at http://localhost:4173/
```

Build the site together with the single-HTML demo at `public/manual.html`:

```bash
scripts/site-build.sh                    # Build CLI, demo, and VitePress site
scripts/site-build.sh preview            # Build and start the preview server
```

## Deployment base path

GitHub or GitLab project pages are served from a subpath such as `https://kuttsun.github.io/monodocs/`. Pass `SITE_BASE` for those deployments. Omit it for a custom domain or user pages, where the default is `/`.

```bash
SITE_BASE=/monodocs/ scripts/site-build.sh
```
